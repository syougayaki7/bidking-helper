# Bid King Helper v2.2

一个给《BidKing：竞拍之王》石油佬角色使用的离线估值网页工具。

## 打开方式

直接用浏览器打开 `index.html`，或者用 VS Code 打开整个文件夹，再用 Live Server 预览。

也可以在本目录运行本地预览服务器：

```powershell
node serve-local.mjs
```

然后打开 `http://127.0.0.1:5173/`。

## 发布到 GitHub Pages

把本目录作为仓库根目录上传到 GitHub。进入仓库的 Settings -> Pages：

- Source 选择 `Deploy from a branch`
- Branch 选择 `main`
- Folder 选择 `/ (root)`
- 保存后等待 GitHub Pages 构建完成

## 当前功能

- 输入总件数和总格数
- 输入红、金、紫、蓝、绿白各自的件数、总格数或均格数；空白代表未知
- 估值使用 `data/items.json` 统计出的品质件均价；红色按“出现概率与价值成反比”的期望值处理
- 已知件数会固定，未知件数会按剩余总件数反推
- 每个颜色可额外输入“至少件数”和已知单件格数列表，例如 `5 6 2 1`
- 均格数按游戏里显示的两位小数处理，反推所有可能的整数件数/格数
- 可能件数/格数会参考数据库里的单件合法格数，并结合已知单件格数列表过滤
- 只有总件数、没有颜色线索时使用快速估算，避免首次打开就枚举大量方案
- 用公式摘要显示红色可能件数，不展开所有方案
- 按官方物品表统计均价估算整体价值
- 价值显示最低、稳妥和期望；稳妥按金以下均值加红色最低价，红色期望价按价值反比概率处理
- `data/items.json` 已导入游戏本地表中的藏品候选，包含 `TypeList` 中带有 `100` 或 `101-110` 的物品

## 发布前检查

当前项目没有 API key、账号信息或联网依赖。发布到 GitHub 前仍建议检查：

- `.env` 或其他密钥文件没有提交
- README 里没有个人本地路径
- 构建产物和临时文件没有提交

## v2 数据源

数据来自本地游戏目录中的 `BidKing_Data/StreamingAssets/Tables/Item.txt`。该文件是 base64 包装的表格文本。

当前导入规则：

- 保留 `TypeList` 包含 `101-110` 的普通藏品
- 保留 `TypeList` 包含 `100` 的特殊藏品
- 排除箱子、消耗型道具、头像、皮肤、角色、称号等奖励/装饰/功能类物品

特殊藏品目前有：

- 金陵折扇
- 双蟾纳宝
